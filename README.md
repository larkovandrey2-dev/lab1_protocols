## Описание задачи
Реализовать подсистему приема задач из разнородных источников (файл, сеть, генератор).
Ключевое условие: источники **не должны** быть связаны наследованием от общего родительского класса. Вместо этого используется **Duck Typing** и контрактное программирование через `typing.Protocol`.

## Выполнение требований

| Требование | Статус | Реализация                                                                                  |
| :--- |:------:|:--------------------------------------------------------------------------------------------|
| **Единый контракт** |   +    | Описан класс `TaskProtocol` с методом `get_tasks()`.                                        |
| **Использование Protocol** |   +    | Контракт наследуется от `typing.Protocol`.                                                  |
| **Runtime-проверка** |   +    | Декоратор `@runtime_checkable` позволяет использовать `isinstance()` для протоколов.        |
| **Отсутствие наследования** |   +    | Источники (`FileSource`, `ApiSource`) не наследуются от протокола, но реализуют его методы. |
| **Структура Task** |   +    | Использован `dataclass` для хранения `id` и `payload`.                                      |
| **Обработка ошибок** |   +    | `FileSource` корректно обрабатывает отсутствие файла.                                       |
| **Тестирование** |   +    | Покрытие тестами > 95% (модуль `unittest`).                                                 |

## Архитектура классов

### 1. Модель данных
*   **`Task`**: Дата-класс, содержащий минимально необходимую информацию (`id`: int, `payload`: dict).

### 2. Контракт (Интерфейс)
*   **`TaskProtocol(Protocol)`**:
    *   Определяет сигнатуру метода `get_tasks(self) -> List[Task]`.
    *   Является `@runtime_checkable`, что позволяет проверять соответствие интерфейсу во время выполнения программы, а не только статическим анализатором.

### 3. Источники задач
Классы не имеют общего предка, но соблюдают контракт:
*   **`FileSource`**:
    *   Читает задачи из JSON-файла.
    *   Обрабатывает `FileNotFoundError`.
*   **`GeneratorSource`**:
    *   Генерирует синтетические задачи "на лету" (используется List Comprehension).
*   **`APISource`**:
    *   Имитирует получение данных из внешнего API (заглушка).

### 4. Менеджер обработки
*   **`process_tasks(source: object)`**:
    *   Принимает **любой** объект.
    *   Внутри выполняет проверку: `if not isinstance(source, TaskProtocol)`.
    *   Если проверка пройдена — вызывает метод `get_tasks()`.


### Тестирование
Проект покрыт unit-тестами. Проверяется корректность генерации, чтение файлов, создание временных файлов (setUp/tearDown) и работа механизма `isinstance` с протоколами.
